== Projet

**Nom** : plantuml-gradle (plugin Gradle pour generer des diagrammes PlantUML via LLM)
**Stack** : Kotlin, Gradle (plugin development), LangChain4j, Ollama/Gemini/OpenAI
**Documentation** : AsciiDoc + PlantUML + Gradle

== Regles Absolues

=== Git

**INTERDICTION FORMELLE** de commit/push/merge sans permission explicite de l'utilisateur.

Commandes autorisees (lecture seule) : `git status`, `git diff`, `git log`, `git show`
Commandes interdites (sauf ordre) : `git add`, `git commit`, `git push`, `git merge`, `git rebase`

=== Tests

L'agent NE DOIT JAMAIS lancer de tests (`./gradlew test`, etc.) lors de la procedure de fin de session SANS permission explicite de l'utilisateur.

=== Securite

* Jamais de secrets en dur dans le code
* Jamais de credentials dans les fichiers de config
* Jamais de tokens API commites dans Git
* Utiliser des references (`keyRef: ${GOOGLE_API_KEY_1}`)

=== Methodologie de Travail

* **1 fichier a la fois** : Modifier un seul fichier avant de passer au suivant
* **Validation immediate** : Compiler apres chaque modification
* **Contexteleger** : Maximum 50k tokens EAGER (~3000 lignes)

== Architecture

=== Structure du Projet

[source,bash]
----
plantuml-gradle/
├── plantuml-plugin/          # Plugin Gradle principal
│   ├── src/main/kotlin/     # Code source Kotlin
│   ├── src/test/            # Tests unitaires + fonctionnels + Cucumber
│   ├── .agents/             # Documentation agent (sessions, archives, references)
│   └── build.gradle.kts     # Build du plugin
├── settings.gradle.kts       # Settings multi-projets
├── AGENTS.md                 # Accroche pour opencode (ce fichier)
├── AGENT.adoc                # Directives completes (AsciiDoc)
└── opencode.json             # Configuration opencode
----

=== Composants Cles

* **PlantumlPlugin** : Point d'entree du plugin Gradle
* **PlantumlManager** : Orchestrateur principal
* **LlmService** : Service d'appel aux LLM (Ollama, Gemini, OpenAI)
* **ApiKeyPool** : Pool de cles API avec rotation et quota tracking
* **ConfigLoader / ConfigMerger** : Chargement et fusion de configuration YAML/properties/env
* **ProcessPlantumlPromptsTask** : Tache Gradle de traitement des prompts
* **ValidatePlantumlSyntaxTask** : Tache Gradle de validation syntaxique
* **GenerateDiagramDocsTask** : Tache Gradle pour le dogfooding

=== Commandes Utiles

[source,bash]
----
# Compiler
./gradlew -p plantuml-plugin compileKotlin

# Tests unitaires
./gradlew -p plantuml-plugin test

# Tests fonctionnels
./gradlew -p plantuml-plugin functionalTest

# Tests Cucumber
./gradlew -p plantuml-plugin cucumberTest

# Tous les checks
./gradlew -p plantuml-plugin check

# Generer la documentation diagrammes
./gradlew -p plantuml-plugin generateDiagramDocs
----

== Gestion du Contexte — CRITIQUE

* **Que faire ensuite ?** → **BACKLOG.adoc** (suivi autoritaire et prioritaire des taches/bugs)
* **TODO.adoc** → pense-bete informel d'idees, PAS un plan, PAS une liste de priorites
* **PROMPT_REPRISE.adoc** → mission de la session en cours et priorites de la prochaine session

== Session Procedure

=== Debut de Session

1. Lire `PROMPT_REPRISE.adoc` pour la mission en cours
2. Lire `.agents/INDEX.adoc` pour les regles absolues
3. Charger les fichiers ESSENTIALS si l'EPIC est concernee

=== Fin de Session

**Mots-cles declencheurs** : "nouvelle session", "je quitte", "session terminee", "a plus tard", "on arrete la"

**Procedure obligatoire en 6 etapes** :

1. **Creer l'archive LAZY** : `.agents/sessions/{N}-{type}-{sujet}.adoc`
2. **Mettre a jour PROMPT_REPRISE.adoc** : Mission N+1 (max 70 lignes)
3. **Mettre a jour SESSIONS_HISTORY.adoc** : Ajouter la session terminee
4. **Mettre a jour INDEX.adoc** : Date, session en cours N+1
5. **Mettre a jour TEST_COVERAGE_ANALYSIS.adoc** (si tests modifies)
6. **Mettre a jour COMPLETED_TASKS_ARCHIVE.adoc**

**Aucune etape ne peut etre marquee OK si le fichier n'a pas ete effectivement modifie et verifie.**

== Conventions de Code

* Kotlin idiomatic, pas de commentaires sauf demande
* Tests : JUnit5 + Cucumber (BDD)
* Configuration : YAML + properties + env vars
* Documentation : AsciiDoc (.adoc) uniquement (plus de .adoc)
* Les fichiers `.adoc` sont reserves a `AGENTS.md` (convention opencode)

== Fichiers EAGER (charges automatiquement)

|===
| Fichier | Role

| `PROMPT_REPRISE.adoc` | Mission session en cours (max 70 lignes)
| `.agents/INDEX.adoc` | Regles absolues + sessions recentes
| `*_ESSENTIALS.adoc` | Contexte metier critique (si EPIC active)
| `AGENT.adoc` | Ce fichier - Directives completes
|===

== Fichiers LAZY (sur demande)

|===
| Fichier | Role

| `*_REFERENCE.adoc` | Architecture complete
| `.agents/sessions/{N}*.adoc` | Archives detaillees par session
| `.agents/archives/COMPLETED_TASKS_ARCHIVE_*.adoc` | Archive des taches terminees
| `SESSION_PROCEDURE.adoc` | Procedure de fin de session
| `AGENT_MODUS_OPERANDI.adoc` | Methodologie LAZY/EAGER
|===

== 🎯 PARTIE 1 — VUE D'ENSEMBLE

=== 1.1 Objectif

Fournir un **cadre de travail structuré** pour collaborer efficacement avec OpenCode sur cette API Spring Boot PlantUML, en évitant :

* ❌ La saturation du contexte (trop de fichiers chargés)
* ❌ Les hallucinations (LLM perdu dans trop d'informations)
* ❌ Le travail en parallèle non coordonné (plusieurs fichiers modifiés simultanément)
* ❌ La perte de contexte entre les sessions
* ❌ Les erreurs de Git (commits non autorisés)
* ❌ Les tests lancés inutilement en fin de session

=== 1.2 Principes Fondamentaux

[cols="1,4", options="header"]
|===
|Principe |Description

|**LAZY/EAGER**
|Séparer l'information critique (toujours chargée) des détails (chargés sur demande). **EAGER** = tableau de bord de voiture. **LAZY** = manuel du propriétaire.

|**1 session = 1 objectif**
|Une session traite UNE seule tâche (1 test, 1 bug, 1 feature). Durée cible : 15-30 minutes.

|**Archive systématique**
|Chaque session terminée est archivée avec résumé et prochaines étapes. Rien ne se perd, tout se retrouve.

|**Git manuel**
|L'agent NE JAMAIS committer sans permission explicite. C'est l'utilisateur qui gère Git.

|**Tests sur demande**
|L'agent NE JAMAIS lancer de tests en fin de session sans demande. Les tests prennent 1-3 minutes.
|===

=== 1.3 Philosophie LAZY/EAGER

**Problème** : Les LLM ont un contexte limité. Charger trop d'informations dilue l'attention et coûte des tokens.

**Solution** : Séparer l'information en 2 catégories :

[cols="2,2,2", options="header"]
|===
|Type |Caractéristiques |Exemples

|**EAGER** (toujours chargé)
|• Critique pour démarrer + < 100 lignes + Lu automatiquement
|`PROMPT_REPRISE.adoc`, `INDEX.adoc` (section règles)

|**LAZY** (sur demande)
|• Détails complets + Illimité + Lu sur demande explicite
|Archives sessions, templates détaillés, références
|===

**Analogie** :
* **EAGER** = Tableau de bord de voiture (vitesse, carburant, alertes)
* **LAZY** = Manuel du propriétaire (détails techniques, historique entretien)

== ⚠️ PARTIE 2 — RÈGLES ABSOLUES

=== 2.1 Commits/Git — INTERDICTION FORMELLE

**L'agent NE DOIT JAMAIS** exécuter de commit, push, merge, ou toute commande git modifiant l'historique **SANS permission explicite de l'utilisateur**.

[cols="1,4", options="header"]
|===
|Autorisé |`git status`, `git diff`, `git log`, `git show` (lecture seule)

|Interdit
|`git add`, `git commit`, `git push`, `git merge`, `git rebase` (sauf ordre explicite)
|===

**Procédure obligatoire avant tout commit** :

1. Montrer les modifications (`git diff --stat`)
2. Demander : `"Veux-tu que je commit ces changements ?"`
3. **Attendre confirmation explicite** (`"oui"`, `"commit"`, `"vas-y"`)
4. **Seulement après** : exécuter le commit

**Exemple de dialogue** :

[source]
----
Utilisateur: "La correction est terminée"

Agent: Voici les modifications :
 M src/main/kotlin/app/plantuml/PlantUmlController.kt (+45 lignes)
 M src/test/kotlin/app/plantuml/PlantUmlControllerTest.kt (+30 lignes)

Veux-tu que je commit ces changements ?

Utilisateur: "Oui"

Agent: [exécute git add -A && git commit -m "feat: create PlantUML convert endpoint"]
----

=== 2.2 Tests en Fin de Session — INTERDICTION FORMELLE

**L'agent NE DOIT JAMAIS** lancer de tests (`./gradlew test`, `./gradlew bootRun`, etc.) lors de la procédure de fin de session **SANS permission explicite de l'utilisateur**.

[cols="1,4", options="header"]
|===
|Autorisé
|Résumer les résultats des tests déjà exécutés pendant la session

|Interdit
|Lancer `./gradlew test` ou tout autre test en procédure de fin de session
|===

**Raison** : Les tests prennent 1-3 minutes et ralentissent inutilement la clôture de session. Si l'utilisateur veut voir les résultats, il/elle le demandera explicitement.

**Procédure de fin de session** :
1. Résumer les réalisations (fichiers créés/modifiés, scénarios passants)
2. Mettre à jour `PROMPT_REPRISE.adoc` pour la session suivante
3. Archiver le contexte dans `.agents/sessions/`
4. **NE PAS lancer de tests** sauf demande explicite

=== 2.3 Secrets et Credentials — SÉCURITÉ MAXIMALE

**INTERDICTION ABSOLUE** :

* ❌ Jamais de secrets en dur dans le code (`secret: secret`)
* ❌ Jamais de credentials dans `application.yaml`
* ❌ Jamais de tokens API commités dans Git

**Solution** :

[source,yaml]
----
# application.yaml
app:
  security:
    jwt:
      secret: ${JWT_SECRET:}  # Variable d'environnement
      base64-secret: ${JWT_BASE64_SECRET:}
----

[source,bash]
----
# .env (ajouté à .gitignore)
JWT_SECRET=MjNlZmQzMzgwM2MzN2Fm...
----

=== 2.4 CORS — Restriction Nécessaire

**Configuration actuelle (trop permissive — à corriger)** :

[source,yaml]
----
# ⚠️ DANGEREUX en production
app:
  cors:
    allowed-origins: '*'
    allowed-methods: '*'
    allowed-headers: '*'
    allow-credentials: true  # ⚠️ Incompatible avec *
----

**Configuration recommandée** :

[source,yaml]
----
# ✅ SÉCURISÉ
app:
  cors:
    allowed-origins:
      - https://app.jhipster.com
      - https://preview.jhipster.com
    allowed-methods:
      - GET
      - POST
      - OPTIONS
    allowed-headers:
      - Authorization
      - Content-Type
    exposed-headers:
      - Authorization
      - Link
      - X-Total-Count
    allow-credentials: true
    max-age: 1800
----

=== 5.3 Règles de Proposition

[source,kotlin]

@Configuration
@EnableWebFluxSecurity
@EnableReactiveMethodSecurity
class SecurityConfiguration {

    @Bean
    fun springSecurityFilterChain(http: ServerHttpSecurity): SecurityWebFilterChain =
        http
            .csrf { it.disable() }  // ✅ Justifié pour API stateless JWT
            .cors { }  // ✅ Configuration CORS externalisée
            .headers { headers ->
                headers.contentSecurityPolicy { csp ->
                    csp.policyDirectives("default-src 'self'")  // ✅ CSP activé
                }
                headers.frameOptions { it.disable() }  // ✅ Nécessaire pour SPA
            }
            .authorizeExchange { exchanges ->
                exchanges
                    .pathMatchers("/api/plantuml/**").permitAll()  // ✅ Endpoints publics
                    .pathMatchers("/api/**").authenticated()  // ✅ Authentification requise
                    .anyExchange().authenticated()
            }
            .build()
}

----
@RestController
@RequestMapping("/api/plantuml")
class PlantUmlController(
    private val plantUmlService: PlantUmlService
) {

    @PostMapping("/convert")
    suspend fun convertToSvg(
        @Valid @RequestBody request: PlantUmlRequest
    ): ResponseEntity<PlantUmlResponse> = try {
        plantUmlService
            .convertToSvg(request.code)
            .map { ResponseEntity.ok(it) }
            .getOrElse { error ->
                when (error) {
                    is InvalidPlantUmlException ->
                        ResponseEntity.badRequest()
                            .body(error.toProblemDetail())
                    is TimeoutException ->
                        ResponseEntity.status(GATEWAY_TIMEOUT)
                            .body(error.toProblemDetail())
                    else ->
                        ResponseEntity.internalServerError()
                            .body(error.toProblemDetail())
                }
            }
    } catch (e: Exception) {
        ResponseEntity.internalServerError()
            .body(e.toProblemDetail())
    }
}

// DTO de requête
data class PlantUmlRequest(
    @NotBlank(message = "Le code PlantUML est obligatoire")
    @Size(max = 10000, message = "Le code ne doit pas dépasser 10000 caractères")
    val code: String,

    @Pattern(
        regexp = "^(svg|png|pdf)$",
        message = "Le format doit être svg, png ou pdf"
    )
    val format: String = "svg"
)

// DTO de réponse
data class PlantUmlResponse(
    val svg: String,
    val warnings: List<String> = emptyList(),
    val processingTimeMs: Long
)
----

== Le saut conceptuel : de consommateur à producteur-documenté

Le plugin `plantuml-gradle` est un **consommateur** de LLM : il prend un prompt textuel et produit un diagramme. L'intégration Graphify en fait aussi un **producteur-documenté** : le code source est transformé en knowledge graph, puis en diagrammes PlantUML par le plugin lui-même.

== Architecture : l'adaptateur GraphifyPromptAdapter

=== Composant unique à ajouter

Un seul fichier Kotlin : `GraphifyPromptAdapter.kt`

[source,kotlin]
----
package plantuml.service

import com.fasterxml.jackson.databind.ObjectMapper
import java.io.File

/**
 * Adapts Graphify knowledge graph subgraphs into .prompt files
 * consumable by ProcessPlantumlPromptsTask.
 *
 * Bridges the gap between structural knowledge (graph.json)
 * and the LLM prompt format expected by the existing pipeline.
 */
class GraphifyPromptAdapter(
    private val graphFile: File,
    private val promptsDir: File
) {

    private val mapper = ObjectMapper()

    data class SubgraphResult(
        val promptFile: File,
        val communityName: String,
        val nodes: List<String>,
        val edges: List<String>
    )

    /**
     * Extract a subgraph by community name and write a .prompt file.
     *
     * The prompt follows the same format as manual .prompt files,
     * so [ProcessPlantumlPromptsTask] processes it identically.
     */
    fun generatePrompt(subgraphName: String): SubgraphResult {
        val graph = mapper.readTree(graphFile)

        // Find community matching subgraphName
        val communities = graph.get("communities")
        val community = communities?.find {
            it.get("name")?.asText()?.contains(subgraphName, ignoreCase = true) == true
        } ?: communities?.first()
            ?: throw IllegalArgumentException("No community found for: $subgraphName")

        val communityName = community.get("name").asText()
        val nodes = community.get("nodes")?.map { it.asText() } ?: emptyList()
        val edges = community.get("edges")?.map { it.asText() } ?: emptyList()

        // Build .prompt content — same format as manual prompts
        val promptContent = buildPrompt(communityName, nodes, edges)

        // Write .prompt file to prompts directory
        val promptFile = promptsDir.resolve("auto-${communityName.replace(" ", "-")}.prompt")
        promptFile.writeText(promptContent)

        return SubgraphResult(promptFile, communityName, nodes, edges)
    }

    /**
     * Generate .prompt files for all communities in the graph.
     */
    fun generateAllPrompts(): List<SubgraphResult> {
        val graph = mapper.readTree(graphFile)
        val communities = graph.get("communities")
            ?: throw IllegalArgumentException("No communities in graph")

        return communities.map { community ->
            val name = community.get("name").asText()
            generatePrompt(name)
        }
    }

    private fun buildPrompt(
        communityName: String,
        nodes: List<String>,
        edges: List<String>
    ): String {
        return """
        |Generate a PlantUML class diagram for the "$communityName" module of the plantuml-gradle plugin.
        |
        |Classes and their relationships:
        |
        |${nodes.joinToString("\n") { "- $it" }}
        |
        |Relationships:
        |
        |${edges.ifEmpty { listOf("(no explicit edges — infer from class names)") }.joinToString("\n") { "- $it" }}
        |
        |Requirements:
        |- Use EXTRACTED relations only (ignore INFERRED)
        |- Include class names with their key attributes
        |- Use proper UML relations: -->, ..>, *--, o--
        |- No style directives, no skinparam, no colors
        |- Structure only, use French for labels where appropriate
        |""".trimMargin()
    }
}
----

=== Points clés de l'adaptateur

[arabic]
. **Même format que les `.prompt` manuels** — Le pipeline existant traite les prompts auto-générés exactement comme les prompts manuels. Aucune modification de `ProcessPlantumlPromptsTask`.
. **Même `LlmService`** — L'appel LLM passe par le même pool de clés API, la même rotation, le même audit. Aucune duplication.
. **Même `DiagramProcessor`** — La boucle itérative (génération → validation → correction) est la même. Si le LLM génère du PlantUML invalide, il est corrigé automatiquement.
. **Même `ValidatePlantumlSyntaxTask`** — La validation est la même, que le prompt vienne d'un fichier manuel ou d'un sous-graphe.
. **Même `PlantumlService`** — Le rendu PNG/SVG est le même.

L'adaptateur ne fait que convertir : `graph.json` → `.prompt` fichier → pipeline existant. **Zéro modification du code existant.**

=== Nouvelle tâche Gradle : `generateDiagramDocs`

[source,kotlin]
----
package plantuml.tasks

import org.gradle.api.DefaultTask
import org.gradle.api.tasks.TaskAction
import org.gradle.work.DisableCachingByDefault
import plantuml.service.GraphifyPromptAdapter
import java.io.File

@DisableCachingByDefault(
    because = "Depends on Graphify output which may change"
)
abstract class GenerateDiagramDocsTask : DefaultTask() {

    init {
        group = "plantuml"
        description = "Auto-generate PlantUML documentation diagrams from Graphify knowledge graph"
    }

    @TaskAction
    fun generateDocs() {
        val graphFile = project.rootDir.resolve("graphify-out/graph.json")
        val promptsDir = project.rootDir.resolve("prompts")

        if (!graphFile.exists()) {
            throw GradleException(
                "graphify-out/graph.json not found. Run /graphify . first."
            )
        }

        promptsDir.mkdirs()

        val adapter = GraphifyPromptAdapter(graphFile, promptsDir)

        val generateAll = project.findProperty("plantuml.diagram.all")?.toString() == "true"
        val subgraph = project.findProperty("plantuml.diagram.subgraph")?.toString()

        val results = if (generateAll) {
            adapter.generateAllPrompts()
        } else {
            listOf(adapter.generatePrompt(subgraph ?: "service layer"))
        }

        results.forEach { result ->
            logger.lifecycle(
                "Generated prompt: {} ({} nodes, {} edges)",
                result.promptFile.name, result.nodes.size, result.edges.size
            )
        }

        logger.lifecycle(
            "Generated {} prompt(s). Run processPlantumlPrompts to render.",
            results.size
        )
    }
}
----

[source,kotlin]
----
@RestController
@RequestMapping("/api/docs")
class DocsController(
    private val llmService: LlmService,      // même que plantuml-gradle
    private val apiKeyPool: ApiKeyPool,        // même rotation
    private val graphifyAdapter: GraphifyPromptAdapter  // même adaptateur
) {

    @PostMapping("/generate")
    fun generateDocs(
        @RequestParam subgraph: String,
        @RequestParam model: String = "qwen3.5-cloud"
    ): DocsResponse {
        // Même pipeline, même ApiKeyPool, même LlmService
        val prompt = graphifyAdapter.generatePrompt(subgraph)
        val result = diagramProcessor.processPrompt(prompt)
        return DocsResponse(result)
    }
}

== Avertissement — Racine de Travail

**Ce fichier est dans `plantuml-plugin/`**, la racine de travail du projet.

La racine du depot git (`plantuml-gradle/`) ne contient QUE le repo git et les fichiers
de bootstrap opencode (`AGENTS.md`, `AGENT.adoc` racine). TOUT le reste vit ici.

* Les chemins dans ce fichier sont relatifs a **`plantuml-plugin/`**
* L'agent travaille depuis **`plantuml-plugin/`**
* Les commandes Gradle s'executent depuis **`plantuml-plugin/`** ou avec `-p plantuml-plugin`

== Projet

**Nom** : plantuml-gradle (plugin Gradle pour generer des diagrammes PlantUML via LLM)
**Stack** : Kotlin, Gradle (plugin development), LangChain4j, Ollama/Gemini/OpenAI
**Documentation** : AsciiDoc + PlantUML + Gradle

== Regles Absolues

=== Git

**INTERDICTION FORMELLE** de commit/push/merge sans permission explicite de l'utilisateur.

Commandes autorisees (lecture seule) : `git status`, `git diff`, `git log`, `git show`
Commandes interdites (sauf ordre) : `git add`, `git commit`, `git push`, `git merge`, `git rebase`

=== Tests

L'agent NE DOIT JAMAIS lancer de tests (`./gradlew test`, etc.) lors de la procedure de fin de session SANS permission explicite de l'utilisateur.

=== Securite

* Jamais de secrets en dur dans le code
* Jamais de credentials dans les fichiers de config
* Jamais de tokens API commites dans Git
* Utiliser des references (`keyRef: ${GOOGLE_API_KEY_1}`)

=== Methodologie de Travail

* **1 fichier a la fois** : Modifier un seul fichier avant de passer au suivant
* **Validation immediate** : Compiler apres chaque modification
* **Contexte leger** : Maximum 50k tokens EAGER (~3000 lignes)

* **PlantumlPlugin** : Point d'entree du plugin Gradle
* **PlantumlManager** : Orchestrateur principal

== Gestion du Contexte — CRITIQUE

* **Que faire ensuite ?** → **`BACKLOG.adoc`** (suivi autoritaire et prioritaire des taches/bugs)
* **`TODO.adoc`** → pense-bete informel d'idees, PAS un plan, PAS une liste de priorites
* **`PROMPT_REPRISE.adoc`** → mission de la session en cours et priorites de la prochaine session

== Session Procedure

=== Debut de Session

1. Lire `PROMPT_REPRISE.adoc` pour la mission en cours
2. Lire `.agents/INDEX.adoc` pour les regles absolues
3. Charger les fichiers ESSENTIALS si l'EPIC est concernee

=== Fin de Session

**Mots-cles declencheurs** : "nouvelle session", "je quitte", "session terminee", "a plus tard", "on arrete la"

**Procedure obligatoire en 6 etapes** :

1. **Creer l'archive LAZY** : `.agents/sessions/{N}-{type}-{sujet}.adoc`
2. **Mettre a jour PROMPT_REPRISE.adoc** — Mission N+1 (max 70 lignes)
3. **Mettre a jour SESSIONS_HISTORY.adoc** : `.agents/SESSIONS_HISTORY.adoc` — Ajouter la session terminee
4. **Mettre a jour INDEX.adoc** : `.agents/INDEX.adoc` — Date, session en cours N+1
5. **Mettre a jour TEST_COVERAGE_ANALYSIS.adoc** (si tests modifies)
6. **Mettre a jour COMPLETED_TASKS_ARCHIVE.adoc** : `.agents/archives/COMPLETED_TASKS_ARCHIVE_{mois}.adoc`

**Aucune etape ne peut etre marquee OK si le fichier n'a pas ete effectivement modifie et verifie.**

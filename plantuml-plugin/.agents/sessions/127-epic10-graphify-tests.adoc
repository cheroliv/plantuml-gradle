=== 2. Découvrir et corriger l'incompatibilité de format

Le format natif de graphify 0.4.29 diffère du format attendu :

| Aspect | Parser attendait | Format réel graphify |
|--------|------------------|---------------------|
| Edges | `edges` | `links` |
| Communautés | `communities` array | entier dans chaque node (`"community": 141`) |
| Node name | `name` | `label` |
| Edge label | `label` | `relation` |
| Edge type | `type` | `confidence` (string) |
| Confidence | `confidence` (double) | `confidence_score` (double) + `weight` |

=== 3. Adapter KnowledgeGraphParser

* 3 modes de parsing : `parseLegacyFormat` (communities array), `parseGraphifyFormat` (nodes/links/community int), `parseFlatFormat` (nodes/edges simples sans communities)
* Résolution ID→label : map `idToLabel` pour convertir les IDs des edges en labels de nodes
* Fix `confidence` string vs double : `isNumber` check avant `asDouble()`

=== 4. Bug fixes découverts

. **Renderer** : `$node.name` → `${node.name}` — string interpolation Kotlin affichait `toString()` au lieu du nom
. **Parser** : résolution ID→label — les edges référençaient les IDs, pas les labels
. **Parser** : `confidence` asDouble sur string retournait 0.0 au lieu de null (Jackson default)

== Lecons apprises

* Le format natif de graphify (v0.4.29) est très différent du format supposé en session 125
* Jackson `asDouble()` sur un text node retourne 0.0 (pas null) — utiliser `isNumber` check
* STRING interpolation Kotlin : `$node.name` → `node.toString() + ".name"`, pas `node.name`
* Les IDs (slugs) et les labels sont deux concepts distincts dans graphify — résolution nécessaire

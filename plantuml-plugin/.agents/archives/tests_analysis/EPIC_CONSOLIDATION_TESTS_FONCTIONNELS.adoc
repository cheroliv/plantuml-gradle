== 🛡️ Principe Non-Négociable : Couverture de Tests

_RÈGLE ABSOLUE_ : Toute optimisation doit _préserver 100% de la couverture fonctionnelle_.

=== Critères de validation obligatoires

Pour chaque test optimisé, vérifier :

|===
| Assertion | Vérification | Exemple
| -----------|--------------|---------
| ✅ **BUILD SUCCESSFUL** | Vérifié dans output | `result.output.contains("BUILD SUCCESSFUL")`
| ✅ **Task exécutée** | TaskOutcome vérifié | `result.task(":task")?.outcome == SUCCESS`
| ✅ **Extension configurée** | Properties vérifiées | `result.output.contains("plantuml")`
| ✅ **Erreurs gérées** | Message d'erreur présent | `result.output.contains("timeout")`
| ✅ **Fichiers créés** | Fichiers existants | `File(...).exists()`
|===

=== Interdictions formelles

* ❌ _Supprimer une assertion_ même "évidente"
* ❌ _Fusionner deux tests_ en un seul (perte de granularité)
* ❌ _Remplacer une assertion_ par un commentaire
* ❌ _Inline du code_ qui supprime des vérifications

=== Exemple de refactorisation acceptable

[source,kotlin]
----
// ✅ ACCEPTABLE : Code inline, mêmes assertions
@Test
fun `should apply plugin successfully`() {
    File(projectDir, "build.gradle.kts").writeText(
        """plugins { id("com.cheroliv.plantuml") }"""
    )
    val result = GradleRunner.create()
        .withProjectDir(projectDir)
        .withArguments("help")
        .build()
    
    // ✅ Mêmes assertions préservées
    assertTrue(result.output.contains("BUILD SUCCESSFUL"))
    assertTrue(result.output.contains("plantuml"))
}
----

=== Exemple de refactorisation interdite

----
// ❌ INTERDIT : Assertion supprimée

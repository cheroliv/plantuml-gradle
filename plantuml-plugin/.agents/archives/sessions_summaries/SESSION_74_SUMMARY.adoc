== 📚 Leçons Apprises

=== Mockito en Kotlin — Bonnes Pratiques

. _Utiliser `mockito-kotlin`_ : La bibliothèque `mockito-kotlin` fournit des extensions Kotlin qui gèrent mieux l'interopérabilité que `org.mockito.Mockito.*`

. _Éviter `any(Class::class.java)`_ : En Kotlin, préférez `any<T>()` ou `any()` avec inférence de type

. _Logger réel vs mock_ : Pour les tests de haut niveau, un vrai logger SLF4J est plus simple qu'un mock

. _`whenever()` vs `when()`_ : `whenever()` évite le conflit avec le mot-clé Kotlin `when`

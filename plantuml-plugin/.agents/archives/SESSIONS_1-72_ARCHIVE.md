## 📋 Leçons Majeures (Sessions 1-72)

### Architecture
- ✅ **Experts Virtuels** : 1 agent avec 6 casquettes contextuelles
- ✅ **Cycle de Mémoire** : N0-N4 (STRATEGIE → AGENTS → WARNINGS → REPRISE → CHECKLISTS)
- ✅ **Architecture hexagonale** : Ports & adapters pour providers LLM

### Tests
- ✅ **WireMock** : Mock des providers LLM pour tests rapides
- ✅ **GradleRunner partagé** : 1 instance pour tous les tests fonctionnels
- ✅ **Nested classes** : Consolidation de 21 fichiers → 1 fichier mère
- ✅ **Tags JUnit5** : `@Tag("quick")` et `@Tag("slow")` pour exécution sélective
- ✅ **testcontainers** : PostgreSQL réel pour tests RAG

### Performance
- ✅ **Tests fonctionnels** : 2m25s → 1m4s (-55%)
- ✅ **Tags quick** : 18 tests < 5s (dév quotidien ~23s)
- ✅ **Configuration cache** : Option supplémentaire (~10s de gain)

### Qualité
- ✅ **Kover 75%** : Seuil de couverture obligatoire
- ✅ **Branch coverage** : 55-57% (objectif 85% en cours)
- ✅ **0 fallback silencieux** : Erreurs explicites en production

### Documentation
- ✅ **Bilingue EN/FR** : README, Contributing, Troubleshooting
- ✅ **7 providers LLM** : Ollama, OpenAI, Gemini, Mistral, Claude, HuggingFace, Groq
- ✅ **Pipeline RAG** : pgvector, embeddings, lifecycle documenté
